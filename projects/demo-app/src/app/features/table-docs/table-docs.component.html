Documentación - Table, Card y Table-Responsive

Tres componentes para mostrar datos tabulares: table-component (tabla HTML estándar), card-component (vista key-value en cards) y table-responsive-component (usa ambos según el breakpoint). Todos reciben headers, items y actions con soporte para iconos y callbacks. La paginación opcional (showPagination, modos client y server) usa el componente independiente paginator-component (también selector legacy table-pagination-bar): texto de rango, selector de tamaño de página y números de página que se adaptan al ancho disponible (ResizeObserver): si caben, muestra todas; si no, primera, vecinos de la actual, última y …. El tope de vecinos se limita con siblingDelta.

1. Table - Básico

Tabla con encabezados y filas. Los headers definen las columnas (key + label). Los items son objetos cuyas keys coinciden con los headers. 

<table-component
  [headers]="headers"
  [items]="items"
></table-component>

2. Table - Con acciones (iconos)

Acciones por fila con iconos. Cada acción emite actionClicked con actionId, item e index. 

<table-component
  [headers]="headers"
  [items]="items"
  [actions]="actionsWithIcons"
  (actionClicked)="onAction($event)"
></table-component>

Última acción: {{ lastAction() }}

3. Table - Acciones con labels y striped

Acciones con texto. actionsPosition ubica la columna de acciones al inicio o final. striped alterna colores de filas.

4. Card - Vista key-value

Una card por cada item. Muestra pares label: valor. Los headers con hiddenOnCard: true no se muestran en la vista card. 

<card-component
  [headers]="headers"
  [items]="items"
  [actions]="actions"
  [actionsPosition]="'end'"
></card-component>

5. Table - Paginación (modo cliente)

Con showPagination y paginationMode="client" el componente rebanará items según pageIndex y pageSize. El texto muestra el rango visible y el total. (paginationChange) actualiza el estado en el padre. El index en actionClicked es el de la página actual; para el índice global usa pageIndex * pageSize + index. 

<table-component
  [items]="manyItems"
  showPagination
  paginationMode="client"
  [pageIndex]="pageIndex()"
  [pageSize]="pageSize()"
  (paginationChange)="onPagination($event)"
/>

6. Table-Responsive - Paginación

Una sola barra de paginación debajo; tabla y cards comparten el mismo subconjunto de filas. Misma API que la tabla.

7. Modo servidor (nota)

Con paginationMode="server" pasas en items solo la página actual y defines totalCount con el total del API. El padre debe cargar datos al cambiar (paginationChange).

8. Table-Responsive (sin paginación)

Muestra tabla en desktop y cards en móvil. breakpoint puede ser sm (640px), md (768px) o lg (1024px). Redimensiona la ventana para ver el cambio. 

<table-responsive-component
  [headers]="headers"
  [items]="items"
  [actions]="actions"
  breakpoint="md"
  (actionClicked)="onAction($event)"
></table-responsive-component>
