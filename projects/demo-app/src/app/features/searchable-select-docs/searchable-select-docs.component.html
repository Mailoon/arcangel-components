Documentación - Searchable Select

El searchable-select-component muestra items que envía el padre. Con multiple el valor del control es un array de value; el panel no se cierra al elegir (salvo que pongas [closeOnSelect] explícito). El campo de búsqueda solo aparece con filterable; el filtrado lo implementa el padre escuchando (filterQueryChange) (debounce 300 ms). La carga de más opciones es tipo lazy load: al hacer scroll cerca del final se emite (loadMore) si hasMore es true; usa loadingMore mientras traes datos. Mismo disparador que el dropdown (ButtonComponent), tipo DropdownItem y soporte CVA.

1. Básico (sin buscar ni lazy)

Sin filterable no hay input de búsqueda. Sin hasMore no hay carga al scroll.

@if (lastSelection()) { Última selección: {{ lastSelection() }} }

2. Lazy load (scroll)

El padre mantiene la ventana acumulada en items. Con [hasMore] y (loadMore) pides el siguiente lote. Mientras cargas, [loadingMore] evita disparos repetidos y muestra “Cargando…”.

3. Filtro delegado al padre

Con filterable se muestra el buscador; cada cambio (tras debounce) emite filterQueryChange. El padre actualiza [items] con su propia lógica.

4. Formulario reactivo (CVA)

[formControl]; el valor es el value del DropdownItem.
