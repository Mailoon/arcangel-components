Documentación - Searchable Select

El searchable-select-component es el control estándar de lista desplegable del kit: usa button-component como disparador, comparte variantes/tamaños/formas con el botón y admite items tipo DropdownItem. Opcionalmente puedes activar búsqueda delegada al padre (filterable), lazy load al scroll y formControl (CVA). Eventos: selectionChange, opened, closed. Cierre al hacer clic fuera, teclado y chevron animado. El dropdown-component sigue disponible en la librería para menús de acciones puros si lo necesitas.

1. Básico

Un select mínimo requiere items (array de objetos con label y opcionalmente value) y placeholder. El evento (selectionChange) emite el item elegido.

@if (selectedItem(); as sel) { Seleccionado: {{ sel.label }} }

2. Variantes de Color

Las mismas variantes del botón: primary, secondary, outline, ghost, danger, warning, success. Afectan solo al trigger. 

<searchable-select-component
  [items]="items"
  variant="primary"
  placeholder="Primary"
></searchable-select-component>

3. Tamaños

size acepta sm, md o lg, igual que el botón. Define el tamaño del trigger. 

<searchable-select-component
  [items]="items"
  size="sm"
  placeholder="Small"
></searchable-select-component>

4. Shapes

shape puede ser rounded, pill o square. Controla la forma del trigger. 

<searchable-select-component
  [items]="items"
  shape="pill"
  placeholder="Pill"
></searchable-select-component>

5. Con iconos

Los items pueden tener iconClass e iconContent (ej. Material Icons). El trigger admite leftIconTemplate o rightIconTemplate. Por defecto muestra un chevron a la derecha que rota al abrir. 

<searchable-select-component
  [items]="itemsWithIcons"
  [leftIconTemplate]="iconHome"
  placeholder="Con icono"
></searchable-select-component>

6. Placement

placement define la posición del panel: bottom-start, bottom-end, top-start, top-end. Útil cuando el control está cerca de bordes. 

<searchable-select-component
  [items]="items"
  placement="bottom-end"
  placeholder="bottom-end"
></searchable-select-component>

7. Full Width

[fullWidth]="true" hace que el trigger ocupe el 100% del ancho del contenedor. 

<searchable-select-component
  [items]="items"
  [fullWidth]="true"
  placeholder="Select ancho completo"
></searchable-select-component>

8. Items con iconos y dividers

Cada item puede tener iconClass, iconContent y divided: true para mostrar un separador antes. Útil para menús con secciones. 

items = [
  { label: 'Perfil' },
  { label: 'Configuración' },
  { label: 'Cerrar sesión', divided: true }
];

9. Disabled state

[disabled]="true" deshabilita el trigger completo. Cada item puede tener disabled: true para deshabilitarlo individualmente. 

<searchable-select-component
  [items]="items"
  [disabled]="true"
  placeholder="Deshabilitado"
></searchable-select-component>

10. Accesibilidad + Tooltip

Usa ariaLabel y tooltip para lectores de pantalla y contexto al pasar el cursor. 

<searchable-select-component
  [items]="items"
  ariaLabel="Menú de opciones accesible"
  tooltip="Abre el menú de opciones"
></searchable-select-component>

11. Custom item template

Con itemTemplate defines cómo se renderiza cada item. La plantilla recibe el item como $implicit (ej. let-item). 

<searchable-select-component
  [items]="items"
  [itemTemplate]="customItemTpl"
></searchable-select-component>

<ng-template #customItemTpl let-item>
  <span class="font-bold">{{ item.label }}</span>
</ng-template>

12. Eventos (selectionChange, opened, closed)

(selectionChange) emite el item elegido. (opened) y (closed) se disparan al abrir y cerrar el panel. Útil para analytics o lógica condicional. 

<searchable-select-component
  [items]="items"
  (selectionChange)="onItemSelected($event)"
  (opened)="onOpened()"
  (closed)="onClosed()"
></searchable-select-component>
