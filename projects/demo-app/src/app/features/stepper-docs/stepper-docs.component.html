Documentación - Stepper (Step By Step)

El stepper-component es un componente de navegación paso a paso con barra de indicadores, botones configurables (Anterior, Siguiente, Cancelar), colores personalizables por estado y diseño responsive. El consumidor controla el contenido de cada paso y el índice actual. El host usa flex: 1 y align-self: stretch para aprovechar el espacio del contenedor padre; el contenido proyectado va en un área con scroll interno para no desbordar el diseño. Coloca el stepper dentro de un padre con altura definida (p. ej. flex flex-col min-h-[400px]) si quieres que el cuerpo del paso ocupe el alto sobrante.

1. Básico

Stepper simple con tres pasos. Usa currentStep (índice 0-based) y emite previousClicked, nextClicked, cancelClicked. El padre controla qué contenido mostrar según el paso actual. 

<stepper-component
  [steps]="steps"
  [currentStep]="currentStep()"
  (previousClicked)="currentStep.set(currentStep() - 1)"
  (nextClicked)="currentStep.set(currentStep() + 1)"
  (cancelClicked)="onCancel()"
></stepper-component>

<div *ngIf="currentStep() === 0">Paso 1...</div>
<div *ngIf="currentStep() === 1">Paso 2...</div>
<div *ngIf="currentStep() === 2">Paso 3...</div>
@if (currentStep() === 0) {

Contenido del paso Información. Completa los datos básicos.

} @if (currentStep() === 1) {

Contenido del paso Detalles. Aquí van los detalles adicionales.

} @if (currentStep() === 2) {

Contenido del paso Confirmar. Revisa y confirma.

}

2. Colores personalizados

inactiveColor, inactiveBorderColor, activeColor, completedColor para personalizar la apariencia.

Paso {{ stepDisplay(currentStepCustom()) }} de {{ steps.length }}.

3. Iconos en steps completados

Cada step puede tener completedIconClass y completedIconContent (ej: Material Icons) para mostrar un icono cuando el paso está completado.

Paso {{ stepDisplay(currentStepIcons()) }} con iconos en steps completados.

4. Templates en botones

Los botones admiten labelTemplate con contexto { stepIndex, totalSteps } para personalizar el texto.

{{ ctx.stepIndex >= ctx.totalSteps - 1 ? 'Finalizar' : 'Siguiente' }}

El botón "Siguiente" cambia a "Finalizar" en el último paso.

5. Responsive y breakpoint

En desktop los steps se muestran en fila horizontal; en móvil en columna. breakpoint puede ser sm (640px), md (768px) o lg (1024px). Redimensiona la ventana para ver el cambio.

Layout horizontal en pantallas ≥768px, vertical en pantallas menores.

showCancelButton="false" oculta el botón Cancelar.
