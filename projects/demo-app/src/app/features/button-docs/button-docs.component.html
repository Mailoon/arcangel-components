Documentación - Button Component

El button-component es un botón reutilizable y accesible que ofrece múltiples variantes de color, tamaños, formas y soporte para iconos. Incluye estados de carga, éxito/error, debounce, modo enlace (href/routerLink) y personalización mediante clases Tailwind. Es ideal para mantener una apariencia consistente en toda la aplicación.

1. Básico + Iconos con Material Icons

Botón con texto e iconos a izquierda o derecha usando plantillas (leftIconTemplate, rightIconTemplate). Útil para acciones con contexto visual como navegación o acciones principales.

2. Variantes de Color

Siete variantes predefinidas para diferentes contextos: primary (acción principal), secondary, outline, ghost, danger (destructivo), warning y success. Se aplican automáticamente al pasar el input variant. 

<button-component label="Primary" variant="primary"></button-component>
<button-component label="Danger" variant="danger"></button-component>

3. Tamaños

Tres tamaños disponibles: sm (pequeño), md (mediano, por defecto) y lg (grande). Controlan el padding y el tamaño de fuente. 

<button-component label="Small" size="sm"></button-component>
<button-component label="Large" size="lg"></button-component>

4. Shapes

Forma de los bordes: rounded (esquinas redondeadas), pill (completamente redondeado) o square (sin redondeo). 

<button-component label="Pill" shape="pill"></button-component>

5. Loading State

Cuando [loading]="true", el botón muestra un spinner y se deshabilita. Ideal para operaciones asíncronas. Puedes vincularlo a una señal o variable reactiva. 

<button-component
  label="Click to load"
  [loading]="isLoading()"
  (clicked)="simulateLoading()"
></button-component>

6. Estados Success/Error

El input state permite mostrar feedback visual: success (verde) o error (rojo). Útil para confirmar acciones como guardar o eliminar. 

<button-component
  label="Success"
  [state]="currentState()"
  (clicked)="setSuccess()"
></button-component>

7. Full Width

Con [fullWidth]="true" el botón ocupa el 100% del ancho del contenedor. Útil en formularios o layouts de columna única. 

<button-component
  label="Full Width Button"
  [fullWidth]="true"
  variant="primary"
></button-component>

8. Square (Icon Only)

Combina [square]="true" con un icono (sin label) para botones de solo icono. Siempre incluye ariaLabel para accesibilidad. 

<button-component
  [square]="true"
  [leftIconTemplate]="iconHome"
  [ariaLabel]="Home button"
></button-component>

9. Accesibilidad + Tooltip

Usa ariaLabel, ariaDescribedBy y tooltip para mejorar la experiencia con lectores de pantalla y dar contexto al usuario al pasar el cursor. 

<button-component
  label="Accessible Button"
  ariaLabel="Save document"
  tooltip="Click to save your changes"
></button-component>

10. Link Mode

Con href el botón se renderiza como <a> y funciona como enlace externo. Usa target="_blank" y rel="noopener noreferrer" para enlaces que abren en nueva pestaña. 

<button-component
  label="External Link"
  href="https://angular.io"
  target="_blank"
  rel="noopener noreferrer"
></button-component>

11. Keyboard Support + Long Press

El botón responde a Enter y Espacio. El evento (longPress) se emite al mantener presionado unos 500ms, útil para acciones secundarias o confirmación. 

<button-component
  label="Press Enter or Space"
  (clicked)="onKeyboardClick()"
  (longPress)="onLongPress()"
></button-component>

12. Debounce (500ms)

debounceTime en milisegundos evita múltiples clics rápidos. Ideal para evitar envíos duplicados de formularios o llamadas repetidas a APIs. 

<button-component
  label="Click rapidly"
  [debounceTime]="500"
  (clicked)="onDebouncedClick()"
></button-component>

13. Custom Label Template

Usa labelTemplate para reemplazar el texto por contenido personalizado (HTML, iconos, etc.). Pasa una referencia a un ng-template. 

<button-component
  [labelTemplate]="customLabel"
  variant="primary"
></button-component>

<ng-template #customLabel>
  <span class="font-bold">Custom Label</span>
</ng-template>

14. Analytics Tracking

El input trackingId permite identificar clics para analytics. El componente registra en consola cuando se hace clic (puedes conectar esto a tu servicio de tracking). 

<button-component
  label="Track this click"
  trackingId="button-analytics-001"
  variant="primary"
></button-component>
